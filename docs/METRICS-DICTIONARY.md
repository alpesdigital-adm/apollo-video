# Metrics dictionary

Product metrics follow source material to first proxy, explicit approval, manual patches, critic/review rounds, asset decisions, reuse and historically estimated time saved. Approval is a recorded final-review state, never an inferred view. Every rate can segment by workspace, format, locale, objective and production mode.

Quality metrics cover technical hard/warning issues, subtitle error, visual collision, semantic/integrity rejection, synthetic lip-sync/identity/pronunciation failure and claim/context incidents with escape rate. Each metric declares baseline, target, direction and regression window.

Operational metrics cover job outcomes/queue time, provider retry/rate/circuit, cost, cache, renders/resources, storage/age, queue saturation, budget alerts, API SLIs/conflicts, webhook delivery and automation calls. UI, REST, SDK and agent are dimensions only; they never alter domain semantics. Sensitive content is prohibited as a metric dimension.
