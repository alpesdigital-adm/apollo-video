# ADR-140 — Release risks, v1 reuse and public defaults

Eleven risks are controlled by executable gates covering phase scope, subjective quality, cost, providers, claim context, synchronization, library governance, UI boundaries, prompt injection, storage growth and automation abuse.

V1 code is never reused by proximity. Characterization plus a v2 contract, zero direct v1 imports and favorable extraction cost are mandatory. Public defaults for F1/F2/F3/F5 are versioned decisions with owners, evidence and tests.
