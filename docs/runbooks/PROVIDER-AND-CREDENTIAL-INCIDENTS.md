# Provider and credential incidents

For a degraded provider, open its circuit, preserve in-flight operations, refresh capability/price/limit metadata and route only compatible new work to a tested fallback. Never mutate the domain brief to accommodate a vendor.

For a compromised credential, activate the client/workspace kill switch, revoke the credential, stop paid operations, rotate replacements, review request/mutation/spend/download anomalies and export redacted audit evidence. Red-team cases include exfiltration, prompt injection, scope escalation and repeated expensive generation.
